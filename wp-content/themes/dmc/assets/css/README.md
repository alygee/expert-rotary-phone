# Структура CSS файлов

CSS файлы разбиты на модули для лучшей организации и поддержки.

## Структура

```
assets/
├── css/
│   ├── base.css              # Основные стили (переменные, базовые классы)
│   ├── responsive/           # Responsive стили по breakpoints
│   │   ├── 7xl.css          # max-width: 1705px
│   │   ├── 6xl.css          # max-width: 1640px
│   │   ├── 5xl.css          # max-width: 1430px
│   │   ├── 4xl.css          # max-width: 1365px
│   │   ├── 3xl.css          # max-width: 1199px
│   │   ├── 2xl.css          # max-width: 1023px
│   │   ├── xl.css           # max-width: 920px
│   │   ├── lg.css           # max-width: 767px
│   │   ├── md.css           # max-width: 599px
│   │   ├── sm.css           # max-width: 480px
│   │   └── xs.css           # max-width: 345px
│   └── README.md            # Этот файл
├── dist/                     # Скомпилированные файлы
├── img/                      # Изображения
└── js/                       # JavaScript файлы
```

## Главный файл

`style.css` в корне темы импортирует все модули в правильном порядке:

1. Базовые стили (`assets/css/base.css`)
2. Responsive стили от больших к маленьким экранам (mobile-first подход)

## Breakpoints

| Имя | Размер | Описание |
|-----|--------|----------|
| 7xl | 1705px | Очень большие экраны |
| 6xl | 1640px | Большие экраны |
| 5xl | 1430px | Средне-большие экраны |
| 4xl | 1365px | Средние экраны |
| 3xl | 1199px | Планшеты (ландшафт) |
| 2xl | 1023px | Планшеты (портрет) |
| xl  | 920px  | Маленькие планшеты |
| lg  | 767px  | Большие мобильные |
| md  | 599px  | Средние мобильные |
| sm  | 480px  | Маленькие мобильные |
| xs  | 345px  | Очень маленькие мобильные |

## Редактирование

- Для изменения базовых стилей редактируйте `assets/css/base.css`
- Для изменения responsive стилей редактируйте соответствующий файл в `assets/css/responsive/`
- Все изменения автоматически подхватываются через импорты в `style.css`

## Примечания

- Все пути к изображениям в CSS файлах должны быть относительно расположения CSS файла (например: `url('../img/image.svg')`)
- Порядок импорта важен - responsive стили идут от больших к маленьким экранам
- Все статические ресурсы (css, js, img, dist) находятся в папке `assets/`
